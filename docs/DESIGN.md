# 设计原则

Tau 当前的“设计”首先指 **产品宿主与交互风格的设计**，而不是先做视觉系统。  
因为当前第一目标是把 `Tau.CodingAgent` 做成一条真实可走通的 CLI 路径。

## 当前设计重心

### 1. CLI-first

当前阶段所有交互设计都服从：

- 编码 Agent 是第一入口
- 终端交互是第一宿主
- `Tau.Tui` 是当前最优先的交互基础设施

### 2. 信息密度高于装饰

无论是 CLI 还是未来 Web UI，都优先保证：

- 状态阶段清楚
- 消息边界清楚
- 工具执行可见
- 错误位置可定位

不优先追求视觉装饰或复杂动效。

### 3. 核心语义先统一，再做多宿主外观

未来不管是：

- `Tau.CodingAgent`
- `Tau.WebUi`
- `Tau.Mom`

都应该共享同一套核心 Agent 语义，而不是每个宿主各做一套行为解释。

## 当前阶段的具体设计约束

### `Tau.CodingAgent`

- 先做清晰、可调试、可扩展的交互结构
- 不追求一开始就 1:1 复制上游全部 UI 细节
- 允许先实现最小输入框、消息区、工具状态区

### `Tau.WebUi`

- 当前仍处于规划态
- 在 CLI P0 未稳定前，不定义复杂视觉规范
- 等进入实现阶段，再单独细化浏览器宿主的组件和样式约束

## 当前不做的事

- 没有真实交互实现时就写完整设计系统文档
- 还没明确宿主边界时，就沉迷于主题、颜色和动效细节
- 为未来可能的前端提前规定大量不存在的组件规范
